# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

`mrgsolve` is an R package for pharmacokinetic/pharmacodynamic (PK/PD) simulation from ODE-based models. Users write models in a domain-specific language (DSL) embedded in `.cpp` or text files, which are parsed, compiled to C++ shared objects, and executed via a LSODA ODE solver.

## Common commands

```bash
make doc          # Regenerate documentation (roxygen2)
make build        # Build package tarball
make install      # Install to ~/Rlibs
make check        # Full check: clean + doc + build + R CMD check + unit tests
make just-check   # Faster check: no tests, no manual
make unit         # Run unit tests in inst/maintenance/unit/
make test-cpp     # Run C++ unit tests in inst/maintenance/unit-cpp/
make test-all     # Run all tests (requires installed package)
make ci           # CI workflow: build + --as-cran check + unit tests
make house        # Rebuild internal model library (inst/base/)
make clean        # Remove build artifacts
```

Run a single test file:
```r
testthat::test_file("tests/testthat/test-foo.R")
```

Set `_MRGSOLVE_SKIP_MODLIB_BUILD_=yes` (already in Makefile) to skip rebuilding the built-in model library during development.

## Architecture

The package has two layers:

**R layer** — user-facing API in `R/`. Key files:
- `mread.R` / `compile.R` — parse DSL blocks, generate C++ code, invoke compiler
- `class_mrgmod.R` — S4 `mrgmod` class (compiled model object with parameters, compartments, init conditions)
- `class_mrgsims.R` — S4 `mrgsims` class (simulation output, data frame + metadata)
- `class_ev.R` — S4 `ev` class (dosing/event specs with dplyr verb support)
- `mrgsim*.R` — simulation runner functions
- `modspec.R` — DSL block parsing
- `matlist.R` — `omegalist`/`sigmalist` matrix containers for OMEGA/SIGMA

**C++ layer** — simulation engine in `src/`:
- `mrgsolve.cpp` — main solver loop, R↔C++ integration via Rcpp
- `LSODA.cpp/h` — ODE solver
- `odeproblem.cpp` — ODE system definition
- `dataobject.cpp` / `datarecord.cpp` — data record management
- `dsl-preprocess.cpp` — DSL preprocessing (fortran-style → C++ syntax)
- `RcppExports.cpp` — auto-generated Rcpp bindings (do not edit manually)

**Model DSL** — users write models with blocks like `$PARAM`, `$ODE`, `$TABLE`, `$MAIN`, `$OMEGA`, `$SIGMA`, `$CAPTURE`, `$PLUGIN`, etc. `compile.R` generates C++ from these blocks. Built-in model examples live in `inst/base/` (built by `inst/maintenance/build_housemodel.R`).

**Tests** — `tests/testthat/` (~60 files) for package-level tests; `inst/maintenance/unit/` for unit tests; `inst/maintenance/unit-cpp/` for C++ layer tests.

## Important conventions

- `Collate:` in `DESCRIPTION` controls R source load order — required for S4 class definitions to load before methods. Run `make doc` (roxygen2) to keep it current.
- `RcppExports.cpp` and `R/RcppExports.R` are auto-generated by `Rcpp::compileAttributes()` — do not edit manually.
- The `_MRGSOLVE_SKIP_MODLIB_BUILD_` env var controls whether `make house` runs during check; set to `no` for CRAN submissions.
- Random effect parameters are named `ETA(n)` and residual error `EPS(n)` in model code — these are reserved.
